# MyAgent 代码质量提升清单

> 审计日期：2026-04-30 | 范围：58 个 `.py` 文件 + 1 个 `.js` 文件

## 严重问题（建议立即修复）

### 1. `router.py` 缺少 `current_provider` 属性

- **文件：** `providers/router.py` vs `providers/router-dsv4.py`
- **问题：** `router-dsv4.py` 比 `router.py` 多了 `current_provider` 属性（第82-88行），但 `router.py` 才是所有模块实际导入的版本。CLI 代码中有 `router.current_provider` 的引用，会导致 `AttributeError`。两个文件已产生代码漂移。
- **方案：** 将 `current_provider` 属性合并进 `router.py`，删除 `router-dsv4.py`。
- **优先级：** P0

```python
# router-dsv4.py 比 router.py 多出的代码段 (line 82-88)
@property
def current_provider(self) -> BaseProvider | None:
    """返回当前可用的主 Provider（列表中第一个未被熔断的）。"""
    for p in self._providers:
        if not self._breaker.is_open(p.name):
            return p
    return self._providers[0] if self._providers else None
```

### 2. `SqliteAuditBackend` 在 async 上下文中同步阻塞事件循环

- **文件：** `observability/backends/sqlite_backend.py`
- **问题：** `flush()` 方法声明为 `async def`，但内部使用同步 `sqlite3.connect()` 执行 I/O。每次 flush 100 条事件时会直接卡住事件循环。且与 `StateStore`（使用 `aiosqlite`）采用了不一致的 SQLite 库。
- **方案：** 统一迁移到 `aiosqlite`，或使用 `asyncio.to_thread()` 包裹同步操作。
- **优先级：** P0

### 3. `HookManager` 静默吞掉所有回调异常

- **文件：** `core/hook.py:113`, `core/hook.py:127`
- **问题：** `emit()` 和 `finalize_content()` 两处 `except Exception: pass`，任何注册的 hook 回调出错都不会有任何日志。开发者排查问题时看不到丝毫线索。
- **方案：** 至少加上 `logger.exception("Hook callback failed", exc_info=True)` 日志记录。
- **优先级：** P0

```python
# core/hook.py:113 — 当前代码
except Exception:
    pass  # ← 需要改为 logger.exception(...)

# core/hook.py:127 — 当前代码  
except Exception:
    pass  # ← 需要改为 logger.exception(...)
```

### 4. Web 前端两个严重错误处理缺口

- **文件：** `web/js/app.js:352`, `web/js/app.js:356`
- **问题：**
  - `ws.onerror` 是空函数（`function () {}`），所有 WebSocket 错误（连接拒绝、TLS 失败）静默丢失。
  - `JSON.parse(event.data)` 没有 try/catch，服务端发来畸形消息时静默崩溃。
- **方案：** 补上 try/catch 和用户可见的错误提示。
- **优先级：** P0

- **方案：** 在 `utils/__init__.py` 中导出 `config`、`logging`、`retry`、`timeout` 模块的公共 API；在 `interfaces/cli/__init__.py` 中导出 `CliUI`、`main`。

### 8. `config/` 目录与 `config.yaml` 命名冲突

- **文件：** 根目录 `config.yaml`（应用配置） vs `config/` 目录（资源文件：`prompt.txt`、`safety_rules.yaml`）

## 工程质量提升

### 10. 测试覆盖率极低

- **现状：** `tests/` 仅有 1 个测试文件（`test_turns.py`，883行），完全覆盖了 Turn 抽象层。但以下模块零测试：
  - `providers/` — Provider 路由、failover、Circuit Breaker
  - `safety/` — 安全规则链、CLIFence 拦截、PolicyEngine 匹配
  - `tools/` — 工具执行器流水线、沙盒、幂等缓存
  - `context/` — 消息管理、Token 预算、状态持久化
  - `observability/` — 审计日志、数据脱敏
- **方案：** 按优先级补齐：safety 链测试 > 工具执行器测试 > Provider 路由 failover 测试 > context 测试。
- **优先级：** P1

### 11. 缺少类型检查

- **问题：** `pyproject.toml` 有 ruff 配置但没有 mypy/pyright。Python 3.12+ 项目若不强制类型检查，类型注解会逐渐退化。
- **方案：** 添加 mypy 配置并逐步启用严格模式。
- **优先级：** P1

```toml
# pyproject.toml 建议补充
[tool.mypy]
strict = true
python_version = "3.12"

[[tool.mypy.overrides]]
module = ["openai.*", "anthropic.*", "aiosqlite.*"]
ignore_missing_imports = true
```

### 12. SQLite PRAGMA 未启用

- **文件：** `context/state.py` — `SQLiteStateStore`
- **问题：** 未启用 WAL 模式、busy timeout、外键约束，并发场景下可能有数据一致性和锁竞争问题。
- **方案：** 在 `initialize()` 方法中添加 pragma 设置。
- **优先级：** P1

```sql
PRAGMA journal_mode=WAL;       -- 读写并发不阻塞
PRAGMA busy_timeout=5000;      -- 锁等待最多 5 秒
PRAGMA foreign_keys=ON;        -- 启用外键约束
```

### 13. `safety/` 中 `async def check()` 实际全同步操作

- **文件：** `safety/cli_fence.py`, `safety/content_rules.py`
- **问题：** `CLIFence.check()`、`InputContentFilter.check()`、`OutputContentFilter.check()` 声明为 `async def` 但内部全是同步正则匹配，产生不必要的协程开销。这是为了适配 `BaseRule` 的 ABC 接口约束。
- **方案：** 考虑将 `BaseRule.check()` 设计为返回 `GuardResult | Awaitable[GuardResult]`，允许同步或异步实现。
- **优先级：** P3

### 14. Web 前端缺少构建工具

- **文件：** `web/` 目录
- **问题：** 纯 HTML+CSS+JS（1409行 CSS + 1126行 JS），没有打包、压缩、lint 工具。随着前端功能增长会成为维护痛点。
- **方案：** 中长期可考虑引入 Vite 或保持 vanilla 但配上 esbuild 压缩。
- **优先级：** P3

### 15. `except Exception:` 静默吞错的其他位置

- **文件：** `interfaces/cli/ui.py:83,93,108,117,131`（5处），`interfaces/websocket/server.py:582`（1处）
- **问题：** CLI UI 中作为降级渲染策略尚可接受，但应至少加 `logger.debug()`。WebSocket 服务器中的 `except Exception` 设置默认标题 "新对话" 可能掩盖真实的会话加载失败。
- **方案：** 加上日志记录。
- **优先级：** P2
