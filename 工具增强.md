# MyAgent 工具增强方案：动态代码加载 & 热注入

## 一、目标

让 MyAgent 支持**运行时动态注册工具**，并且新工具**即刻对活动会话可见**——无需重启 Agent，无需重建 Session。

具体包括：
1. 用 `@make_tool` 装饰器把任意 async 函数一键变成 `BaseTool`，无需手写类
2. 从代码文本 (`exec`) 动态加载工具，支持对话中临时注入
3. 从 Python 模块路径加载工具 (`entry_point`)
4. 修复 `Session/AgentLoop` 中的 schema 快照问题，实现热加载

## 三、核心原理：Python 自省 → JSON Schema

### 3.1 链路

```
Python 函数对象
  ├── __name__         → name
  ├── __doc__          → description
  ├── __annotations__  → 类型信息（get_type_hints）
  ├── __defaults__     → 参数默认值
  └── inspect.signature() → 参数列表 + 是否必填
           │
           ▼
      generate_schema(func)
           │
           ▼
  JSON Schema (与 MCP inputSchema / OpenAI parameters 一致)
```

### 3.2 类型映射

| Python 类型 | JSON Schema type |
|-------------|-----------------|
| `str` | `"string"` |
| `int` | `"integer"` |
| `float` | `"number"` |
| `bool` | `"boolean"` |
| `list` | `"array"` |
| `dict` | `"object"` |

### 3.3 示例

输入：
```python
async def web_search(query: str, max_results: int = 5) -> str:
    """搜索互联网。

    Args:
        query: 搜索关键词
        max_results: 最大返回条数
    """
```

输出：
```json
{
    "type": "object",
    "properties": {
        "query":        {"type": "string",  "description": "搜索关键词"},
        "max_results":  {"type": "integer", "description": "最大返回条数", "default": 5}
    },
    "required": ["query"]
}
```

## 四、模块详解

### 4.1 `schema.py` — JSON Schema 生成器

纯标准库实现，零外部依赖。

```python
from myagent.tools.schema import generate_schema, extract_description

# 自动生成完整 JSON Schema
schema = generate_schema(my_async_func)

# 提取 docstring 第一行作为描述
desc = extract_description(my_async_func)
```

**核心函数：**
- `generate_schema(func, include_descriptions=True)` — 从函数签名生成 JSON Schema
- `extract_description(func)` — 提取 docstring 第一行

**关键设计：**
- 跳过 `self`/`cls` 参数
- 识别默认值判断参数是否必填
- 从 docstring 的 `Args:` 段落提取参数描述（支持 Google 和 Sphinx 风格）
- `get_type_hints()` 失败时回退到 `__annotations__`

### 4.2 `wrapper.py` — 函数 → BaseTool 包装器

```python
from myagent.tools.wrapper import make_tool, FunctionTool

# 用法 1: 装饰器
@make_tool
async def web_search(query: str, max_results: int = 5) -> str:
    """搜索互联网。"""
    return ...

# 用法 2: 直接包装
tool = make_tool(web_search, name="search", description="搜索")

# 用法 3: 覆盖名称
@make_tool(name="translate_text")
async def translate(text: str, target: str = "en") -> str:
    """翻译文本。"""
    ...
```

**FunctionTool 特性：**
- 自动从函数自省生成 `name` / `description` / `parameters_schema`
- 如果函数返回 `ToolResult`，直接使用；否则自动包装为 `ToolResult(content=str(result))`
- 完全兼容现有的 `ToolRegistry` / `ToolExecutor` / `SafetyGuard` 流水线

### 4.3 `loader.py` — 三通道工具加载器

```python
from myagent.tools.loader import ToolLoader
```

#### 通道 A: 模块入口点

```python
tool = ToolLoader.from_entry("myagent.extra.search:web_search")
# 或指定覆盖名
tool = ToolLoader.from_entry("myagent.extra.search:web_search", name="custom_search")
```

`entry` 格式为 `"module.path:attr_name"`：
- `importlib.import_module(module_path)` → 获取模块
- `getattr(module, attr_name)` → 获取目标对象
- 函数 → `FunctionTool` 自动包装
- `BaseTool` 子类 → 直接实例化

#### 通道 B: 代码文本动态加载

```python
tool = ToolLoader.from_code("""
async def add(a: int, b: int) -> str:
    \"\"\"加法运算。\"\"\"
    return str(a + b)
""")
```

原理：`exec(code, namespace)` → 在命名空间中执行代码 → 从 namespace 提取函数对象 → `FunctionTool` 包装。

**自动发现：** 如果代码中只有一个 async 函数，无需指定 `function_name`；否则可通过参数指定。

**安全检查（safe_mode=True，默认启用）：**
- AST 级别拦截危险模块导入（`os`, `subprocess`, `sys`, `shutil`, `socket` 等）
- 语法错误先行检查

#### 通道 A 变体: 代码文件

```python
tool = ToolLoader.from_file("./my_tools/search.py", function_name="web_search")
```

读取文件内容后走 `from_code` 通道。

#### 批量配置驱动加载

```python
configs = [
    {"type": "entry", "entry": "myagent.extra.search:web_search"},
    {"type": "code",  "code": "async def greet(name: str) -> str: ..."},
    {"type": "file",  "path": "./my_tools/analyzer.py", "function": "analyze"},
]
tools = ToolLoader.from_config(configs)
```

## 五、热加载：解决 Schema 快照问题

### 5.1 问题根因

```
Session.__init__()
  L84: self._tool_schemas = executor.get_tool_schemas()  ← 一次性快照
  L92: AgentLoop(..., tool_schemas=self._tool_schemas)    ← 传入快照

AgentLoop.__init__()
  L610: self._tool_schemas = tool_schemas                 ← 保存快照

AgentLoop._create_turn(MODEL)
  L626: ModelTurn(..., tool_schemas=self._tool_schemas)   ← 传入快照
```

调用链：`agent.add_tool() → ToolRegistry.register()` 注册了新工具，但 Session/AgentLoop/ModelTurn 持有的都是构造时的 snapshot，永远不会更新。

### 5.2 修复方案

**3 处、共 7 行改动：**

1. **`session.py`** — 移除预计算 `_tool_schemas`，不再传入 `AgentLoop` 构造函数
2. **`loop.py` `AgentLoop.__init__()`** — 移除 `tool_schemas` 参数和属性
3. **`loop.py` `_create_turn()`** — 每次创建 ModelTurn 时从 `self._executor.get_tool_schemas()` 动态获取

### 5.3 修复后效果

```
agent.add_tool(new_tool)     → ToolRegistry.register(new_tool)       ✓ 立即可注册
session.run("...")           → AgentLoop._create_turn(MODEL)
                               → ModelTurn(tool_schemas=executor.get_tool_schemas())
                               → executor.get_tool_schemas()
                               → registry.list_tools()               ← 动态获龋
                               → LLM 看到包含 new_tool 的最新列表     ✓ 热加载生效
```

每次 ReAct 循环新建 `ModelTurn` 时都会重新获取工具列表，新注册的工具自动对下一次 LLM 调用可见。

## 六、使用场景

### 场景 1：启动时注册预定义工具

```python
from myagent import Agent
from myagent.tools import make_tool

@make_tool
async def web_search(query: str, max_results: int = 5) -> str:
    """搜索互联网"""
    ...

agent.add_tool(web_search)
agent.run("查一下今天天气")
```

### 场景 2：对话中热注入代码工具

```python
from myagent.tools import ToolLoader

code = """
async def analyze_log(path: str, pattern: str = "ERROR") -> str:
    \"\"\"分析日志文件。

    Args:
        path: 日志文件路径
        pattern: 匹配模式
    \"\"\"
    with open(path) as f:
        lines = [l for l in f if pattern in l]
    return f"找到 {len(lines)} 条匹配记录"
"""

# 在对话进行中注入
tool = ToolLoader.from_code(code)
agent.add_tool(tool)
# 下一次 agent.run(...) 自动看到 analyze_log 工具
```

### 场景 3：配置文件驱动批量加载

```yaml
# config.yaml
agent:
  tools:
    default_timeout: 30.0
    extra:
      - type: "entry"
        entry: "myagent.extra.search:web_search"
      - type: "entry"
        entry: "myagent.extra.translator:translate"
        name: "translate_text"
      - type: "file"
        path: "./my_tools/analyzer.py"
        function: "analyze_data"
```

在 `AgentFactory._build_tool_registry()` 中接入：

```python
tools_cfg = self._app_config.get("tools", {})
extra = tools_cfg.get("extra", [])
if extra:
    for tool in ToolLoader.from_config(extra):
        tool_registry.register(tool)
```

## 七、安全机制

| 层级 | 机制 | 说明 |
|------|------|------|
| **代码注入防护** | AST 解析 + 模块白名单 | `loader.py` 的 `_validate_code_safety()` 拦截 `os`/`subprocess`/`sys` 等危险导入 |
| **语法检查** | `ast.parse()` 先行验证 | 语法不合法直接拒绝 |
| **执行隔离** | 与 `CLITool` 隔离 | 动态工具不共享沙盒，执行环境独立 |
| **已有安全流水线** | 复用 `SafetyGuard` | 所有工具（包括动态加载的）统一走 PolicyEngine → CLIFence → ContentFilter |

## 八、与现有架构的集成点

```
ToolLoader.from_entry()/from_code()/from_file()
        │
        ▼
FunctionTool (BaseTool 子类)   ← 与 CLITool、FileReadTool 平级
        │
        ▼
agent.add_tool() → ToolRegistry.register()
        │
        ▼
ToolExecutor (SafetyGuard → Idempotency → Secret → Execute)
        │
        ▼
AgentLoop._create_turn(MODEL) → executor.get_tool_schemas() ← 动态获取
        │
        ▼
ModelTurn → LLM function calling
```

没有任何突破架构的地方——新工具与内置工具走完全相同的流水线。
