# 工具窗口 UI 重构 — 仿 Kimi 样式

## 目标

将当前的工具调用显示从 **带边框的卡片/翻页式 Chip** 改为 **Kimi 风格的垂直时间线列表**，特点：

1. 每个工具调用显示为一行紧凑条目，前面有绿色状态圆点
2. 工具类型标签加粗 + 简短命令/参数描述以浅色显示
3. 去掉翻页机制，所有工具调用同时可见（类似时间线）
4. 展开/折叠详情保留，但默认收起
5. 执行中的工具用 spinner 动画替代绿点

## 当前实现 vs 目标对比

| 特性 | 当前实现 | Kimi 目标 |
|------|---------|----------|
| 布局 | 带边框卡片 (`.tool-chip`)，翻页只显示一个 | 无边框列表，所有项同时可见 |
| 状态指示 | 右侧文字标签 ("执行中"/"完成") + spinner | 左侧绿色圆点 (完成=实心绿, 执行中=脉冲动画) |
| 工具名 | `#1 tool_name` 加 emoji icon | `● Bash` / `● Read` / `● Process` 样式，类型加粗 |
| 参数预览 | 隐藏在展开面板中 | 同一行以灰色字显示命令摘要 |
| 展开面板 | 默认展开当前工具 | 默认全部收起 |
| 翻页 | ◀ 1/3 ▶ 翻页控件 | **无翻页，全部列表可见** |

#### 2.3 新增辅助函数

```javascript
// 获取工具显示标签（中文化 + 简化）
function getToolLabel(toolName) {
    const name = (toolName || "").toLowerCase();
    if (name.includes("cli") || name.includes("command") || name.includes("exec") || name.includes("bash"))
        return "Bash";
    if (name.includes("read") || name.includes("file_read"))
        return "Read";
    if (name.includes("write") || name.includes("file_write"))
        return "Write";
    if (name.includes("search"))
        return "Search";
    if (name.includes("web") || name.includes("http"))
        return "Web";
    return toolName;  // 无法识别则原样显示
}

// 获取命令摘要（单行预览）
function getCommandSummary(toolName, args) {
    if (!args) return "";
    // CLI 类工具显示命令
    if (typeof args === "object") {
        if (args.command) return args.command;
        if (args.cmd) return args.cmd;
        if (args.path) return args.path;
        if (args.file_path) return args.file_path;
        if (args.query) return args.query;
        if (args.url) return args.url;
    }
    // 兜底：JSON 截断
    const str = typeof args === "string" ? args : JSON.stringify(args);
    return truncateStr(str, 80);
}
```
