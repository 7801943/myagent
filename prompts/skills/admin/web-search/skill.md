# Web 检索与资料核验指南

## 适用场景

- 用户要求最新、今天、最近、当前状态、价格、法规、版本、赛程等可能变化的信息。
- 用户需要来源、链接、直接引用、官网资料或精确出处。
- 本地知识不确定，或主题较新、较窄、容易过期。

## 检索策略

- 优先使用官方文档、原始公告、论文、标准、政府或项目主页等一手来源。
- 新闻类问题优先比较发布时间和事件发生时间。
- 技术问题优先查官方文档、源码仓库、发布说明或规范文件。
- 多来源冲突时，明确指出差异，并优先采用权威或更新的来源。

## 回答规范

- 给出结论时附上来源链接。
- 少量引用即可，避免长段复制原文。
- 对推断和来源事实分开表述。
- 如果检索不到可靠依据，直接说明不确定性和已检查的方向。

## 安全边界

- 医疗、法律、金融等高风险建议必须提醒用户咨询专业人士。
- 不把未经核验的网页内容包装成确定事实。
